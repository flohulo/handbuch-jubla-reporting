# 1. Repository forken

Durch einen **Fork** erstellst du eine eigene Kopie des jubla-reporting Codes auf deinem GitHub-Account. So kannst du das Tool für deine Schar individuell anpassen, ohne das Original zu verändern.

---

## Schritt-für-Schritt

1. Öffne das Repository: [github.com/flohulo/jubla-reporting](https://github.com/flohulo/jubla-reporting)
2. Klicke oben rechts auf **«Fork»**
3. Wähle deinen GitHub-Account als Ziel
4. Benenne das Repository nach deiner Schar, z.B. `jubla-wald-reporting`
5. Klicke auf **«Create fork»**

!!! info "Warum forken?"
    Ein Fork ist nötig, damit du:

    - eigene Umgebungsvariablen und Konfigurationen hinterlegen kannst
    - Updates aus dem Original-Repo jederzeit einspielen kannst (`Sync fork`)
    - eine eigene Netlify-Instanz betreiben kannst

---

## Updates einspielen

Wenn es eine neue Version von jubla-reporting gibt, kannst du dein geforktes Repo synchronisieren:

1. Öffne dein geforktes Repo auf GitHub
2. Klicke auf **«Sync fork»** → **«Update branch»**

!!! warning "Eigene Anpassungen prüfen"
    Wenn du Dateien wie `config.js` angepasst hast, prüfe nach dem Sync, ob deine Änderungen noch vorhanden sind.
